- CS, 알고리즘의 꽃이라고 불리웁니다.
- 특성상 정렬은 불가능합니다.
- 키-값 쌍으로 이루어져 있습니다.
- O(1)의 time complexity로 탐색, 추가, 삭제가 가능합니다.
  -> array의 경우 탐색을 위해 O(n)의 시간복잡도를 갖지만
  -> 해시테이블은 해시값을 통해 정해진 idx값으로 index array에 연결된 linked list에 접근하는데
     linked list에 같은 idx로 연결된 키-값들이 있다 하여도 키를 통해 값을 빠르게 찾을 수 있으며
     이는 linked list가 한 번에 끝나게 되면 O(1)이, linked list의 한쪽에 걸린 경우 O(n)가 됩니다.
     (언어가 제공해주는 해시맵, 해시테이블을 사용하는 경우 O(n)과 같은 worst case는 나오지 않습니다.
     따라서 그냥 해시테이블은 O(1)의 타입컴플렉스를 갖는다고 알면 됩니다.)
